Java IO之字符流

发表于 2018-06-12 | 分类于 Java |

字符流操作的数据单元是字符，操作方式与字节流几乎一样

Reader

Reader是所有字符输入流的抽象基类，通过子类来创建实例

常用方法

int read()    //从输入流中读取单个字符，返回读取的字符数据
int read(char[] cbuf)    //从输入流中最多读取cbuf.length个字符的数据，并存储在cbuf中
int read(char[] cbuf, int off, int len)    //从输入流中最多读取cbuf.length个字符数据，从数组的off位置开始存入cbuf

读取文本内容

采用FileReader来读取文本内容

Writer

常用方法

void write(int c)    //将指定的字节/字符输出到输出流，c既可为字节，也可为字符
void write(char[] cbuf)    //将字符数组中的数据输出到指定输出流
void write(char[] cbuf, int off, int len)    
void write(String str)    //将str字符串里包含的字符输出到指定输出流
void write(String str, int off, int len)    //将str从off开始，长度为len的字符输出到指定输出流
void flush()
void close()

直接输出字符串内容

FileWiter写的数据太久不缓冲可能会造成数据缺失
所以写一定数据量后立马调用flush()

处理流的用法

处理流可对外提供更方便的输入输出方法,使用处理流来包装节点流，程序通过处理流来执行输入输出功能。下面使用处理流来包装OutputStream

通常如果需要输出文本内容，都应该讲输出流包装成PrintStream后进行输出
使用处理流包装了底层节点流之后，关闭输入/输出流资源时，只要关闭最
上层的处理流即可，系统会自动关闭被包装的节点流
